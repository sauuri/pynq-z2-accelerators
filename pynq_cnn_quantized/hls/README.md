# HLS Implementation Guide

This directory is reserved for HLS (High-Level Synthesis) artifacts generated from the quantized CNN model.

---

## Two Recommended Paths

### Path A: hls4ml (Keras/ONNX → HLS C++ → Vivado)

hls4ml translates neural network models directly into synthesizable HLS C++.

**Steps:**

```bash
# 1. Export ONNX from host/inference.py
python host/inference.py --export-onnx
# -> checkpoints/quantized_model.onnx

# 2. Install hls4ml
pip install hls4ml[ONNX]

# 3. Convert ONNX to HLS project
python - <<'EOF'
import hls4ml
import onnx

model_onnx = onnx.load('checkpoints/quantized_model.onnx')

config = hls4ml.utils.config_from_onnx_model(
    model_onnx,
    granularity='name',
    default_precision='ap_fixed<16,6>',
    default_reuse_factor=1,
)

hls_model = hls4ml.converters.convert_from_onnx_model(
    model_onnx,
    hls_config=config,
    output_dir='hls/hls4ml_output',
    part='xc7z020clg400-1',   # PYNQ-Z2 FPGA part
    clock_period=10,           # 100 MHz
)

hls_model.compile()
EOF

# 4. Open in Vivado HLS and synthesize
# cd hls/hls4ml_output
# vivado_hls -f build_prj.tcl

# 5. Integrate into Vivado block design
# Add AXI-Stream DMA + custom IP
# Generate bitstream (.bit + .hwh)
```

**Recommended precision for PYNQ-Z2:**
- Weights: `ap_fixed<8,3>` (INT8 quantized)
- Activations: `ap_fixed<16,6>`
- Reuse factor: 1–4 (trade-off: latency vs. resource usage)

---

### Path B: FINN (Brevitas QAT → FINN → bitstream)

FINN (Fast and Flex Neural Network) by Xilinx Research supports fully quantized (binary/int) networks with streaming dataflow.

**Steps:**

```bash
# 1. Retrain with Brevitas (QAT-aware quantization)
pip install brevitas

# Modify model.py to use Brevitas QuantConv2d / QuantLinear
# Then retrain and export:
# python train/train_brevitas.py

# 2. Export to QONNX format
# brevitas.export.export_qonnx(model, torch.randn(1,3,32,32), 'model.onnx')

# 3. Run FINN compiler
# docker pull xilinx/finn
# docker run --rm -v $(pwd):/workspace xilinx/finn \
#     finn build_dataflow model.onnx --platform pynq-z2

# 4. Flash generated bitstream to PYNQ board
# scp output/bitfile.bit xilinx@pynq:/home/xilinx/
```

**When to use FINN vs hls4ml:**
| | hls4ml | FINN |
|---|---|---|
| Model type | FP32 / low-bit | INT2–INT8 (Brevitas) |
| Throughput | Medium | High (streaming dataflow) |
| Ease of use | Easier | More setup required |
| PYNQ support | ✅ | ✅ |

## Resource Estimates for PYNQ-Z2 (xc7z020)

| Resource | Available | Estimated Usage |
|---|---|---|
| LUTs | 53,200 | ~30,000–45,000 |
| FFs | 106,400 | ~20,000–40,000 |
| BRAM | 140 | ~60–100 |
| DSPs | 220 | ~80–150 |

Note: Actual usage depends on precision and reuse factor.
Use `ap_fixed<8,3>` weights to reduce DSP usage significantly.
